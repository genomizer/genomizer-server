trim¶

usage: pyicos [-h] [-v] [-o] [-f EXPERIMENT_FORMAT] [--debug] [--no-sort]
              [--force-sort] [--silent] [--disable-cache] [--keep-temp]
              [--postscript] [--showplots] [--label1 LABEL1] [--label2 LABEL2]

              [--trim-proportion TRIM_PROPORTION] [-p SPECIES]
              [--duplicates DUPLICATES] [--poisson-test POISSON_TEST]
              experiment output

Pyicoller, part of the Pyicoteo suite, is a peak caller for the analysis of
punctuated ChIP-Seq data.

positional arguments:
  experiment            The experiment file or directory
  output                The output file or directory

optional arguments:
  -h, --help            show this help message and exit
  -v, --version         show program's version number and exit
  -o, --open-experiment
                        Defines if the experiment is half-open or closed
                        notation. [Default False]
  -f EXPERIMENT_FORMAT, --experiment-format EXPERIMENT_FORMAT
                        The format the experiment file is written as. The
                        options are ('eland', 'bed', 'bed_wig', 'bed_pk',
                        'bed_spk', 'sam', 'bam', 'counts'). [Default pk]
  --debug
  --no-sort             Force skip the sorting step. WARNING: Use only if you
                        know what you are doing. Processing unsorted files
                        assuming they are will outcome in erroneous results
  --force-sort          Force the sorting step
  --silent              Run without printing in screen
  --disable-cache       Disable internal reading cache. When Clustering low
                        coverage files, it will increase speed and improve
                        memory usage. With very read dense files, the speed
                        will decrease.
  --keep-temp           Keep the temporary files
  --postscript          get the output graphs in postscript format instead of
                        .png
  --showplots           Show the plots as they are being calculated by
                        matplotlib. Note that the execution will be stopped
                        until you close the window pop up that will arise
  --label1 LABEL1       Manually define the first label of the graphs.
  --label2 LABEL2       Manually define the second label of the graphs.
  --tempdir TEMPDIR     Manually define the temporary directory where Pyicoteo
                        will write. By default Pyicoteo will use the temporary
                        directory the system provides (For example, /tmp in
                        unix systems)
  --samtools            Use samtools for reading BAM files [Default: Pyicoteo
                        uses its own library] (reading BAM works without
                        samtools for convert, extend, and other operations,
                        but not for enrichment yet)]
  --skip-header         Skip writing the header for the output file. [Default
                        False]
  --control CONTROL     The control file or directory
  --control-format CONTROL_FORMAT
                        The format the control file is written as. [default:
                        The same as experiment format]
  --open-control        Define if the region file is half-open or closed
                        notation. [Default closed]
  --blacklist BLACKLIST
                        Reads a bed file with coordinates that you want to
                        exclude from the analysis. Useful for discarding
                        "noisy" probable artifactual regions like centromeres
                        and repeat regions. [Default None]
  -O, --open-output     Define if the output is half-open or closed notation.
                        [Default closed]
  -F OUTPUT_FORMAT, --output-format OUTPUT_FORMAT
                        Format desired for the output. You can choose between
                        ('eland', 'bed', 'bed_wig', 'bed_pk', 'bed_spk',
                        'sam', 'bam', 'counts'). WARNING, for some operations,
                        some outputs are not valid. See operation help for
                        more info. [default pk]
  -x FRAG_SIZE, --frag-size FRAG_SIZE
                        The estimated inmmunoprecipitated fragment size. This
                        is used by Pyicoteo to reconstruct the original signal
                        in the original wet lab experiment.
  --round               Round the final results to an integer
  --wig-label WIG_LABEL
                        The label that will identify the experiment in the WIG
                        tracks.
  --span SPAN           The span of the variable and fixed wig formats
                        [Default 40]
  --no-subtract         Don't subtract the control to the output, only
                        normalize.
  --remlabels REMLABELS
                        Discard the reads that have this particular label.
                        Example: --discard chr1 will discard all reads with
                        chr1 as tag. You can specify multiple tags to discard
                        using the following notation --discard chr1 chr2 tagN
  --p-value P_VALUE     The threshold p-value that will make a cluster
                        significant. [Default 0.01]
  --k-limit K_LIMIT     The k limit Pyicoteo will analize to when performing a
                        poisson test. Every cluster that goes over the
                        threshold will have a p-value of 0, therefore
                        considered significant. For performance purposes,
                        raising it will give more precision when defining low
                        p-values, but will take longer to execute. [Default
                        400]
  --correction CORRECTION
                        This value will correct the size of the genome you are
                        analyzing. This way you can take into consideration
                        the real mappable genome [Default 1.0]
  --trim-proportion TRIM_PROPORTION
                        Fraction of the cluster height below which the peak is
                        trimmed. Example: For a cluster of height 40, if the
                        flag is 0.05, 40*0.05=2. Every cluster will be trimmed
                        to that height. A position of height 1 is always
                        considered insignificant, no matter what the cluster
                        height is. [Default 0.3]
  -p SPECIES, --species SPECIES
                        The species that you are analyzing. This will read the
                        length of the chromosomes of this species from the
                        files inside the folder "chrdesc". If the species
                        information is not known, the filtering step will
                        assume that the chromosomes are as long as the
                        position of the furthest read. [Default hg19]
  --duplicates DUPLICATES
                        The number of duplicated reads accept will be counted.
                        Any duplicated read after this threshold will be
                        discarded. [Default 0]
  --poisson-test POISSON_TEST
                        Decide what property of the cluster will be used for
                        the poisson analysis. Choices are ('height',
                        'numtags', 'length') [Default height]
